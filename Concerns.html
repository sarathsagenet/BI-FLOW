Concerns

  • Not informed about new reporting requirements: We are only asked to provide access to SAP data.
    • If the requirements are discussed with us, it might be something that can be done directly in SAP without any need of the additional data movement between systems.
  • No control of what SAP data is extracted or the timings when they are extracted
    • Only the BI related calculation view access is controlled
    • Non-BI extractions from SAP system are not controlled or monitored
    • S/4 system and BI modeling are done on the same system. Means the memory is shared. Even if it is a non-BI extraction, it will affect the BI report performance
  • GMA Report models: The GMA is the biggest model we have in BI. Our request is to re-build this model with CDS views
    • This will make all the GMA reports real-time. Now the data loads run every night
    • The dependency on SAP DataServices (BODS) can be removed, and we will no longer need this system
    • Currently the daily load takes up a lot of memory. On some days it errors out with memory error
    • The Executive Dashboards also use GMA model. Redesigning the GMA model will help to make the Executive dashboards a realtime report
  • Additional Resource in the team
    • Having an additional resource will help us to simultaneously take care of daily maintenance, working on bug fixes, documentation and most importantly, putting in our time in to the new advanced features and configurations available in SAC
    • The GMA remodeling can be completed.
